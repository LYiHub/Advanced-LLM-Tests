# 任务：实现一个多角色权限计算器（Web 全栈）

## 项目概述

请实现一个运行在本地的 Web 计算器应用，不同角色的用户看到和能使用的功能不同。
后端用 Python 标准库（http.server + sqlite3），前端用纯 HTML/CSS/JavaScript，不允许使用任何第三方包（包括 Flask、React 等）。

启动方式：
```bash
python server.py
# 浏览器访问 http://localhost:8080
```

---

## 角色与权限设计

系统存在四种固定角色，权限严格分层：

| 角色 | 可用运算 | 管理能力 |
|------|---------|---------|
| **游客（guest）** | 加法、减法 | 无 |
| **会员（member）** | 加、减、乘、除 | 无 |
| **管理员（admin）** | 加、减、乘、除 | 升级/降级用户角色（不能操作超管） |
| **超级管理员（superadmin）** | 全部运算 | 全部管理能力 + 篡改数学规则 |

**超级管理员默认账号**：用户名 `root`，密码 `root123`，系统初始化时自动创建。

## 功能需求

### 1. 用户认证

- 登录页：输入用户名 + 密码
- 注册页：新注册用户默认为 guest 角色
- 登出功能
- 密码需哈希存储（使用 `hashlib`），不能明文

### 2. 计算器界面

- 登录后展示计算器
- 界面上**只显示当前角色有权限使用的运算按钮**
  - guest：只有 + 和 - 按钮
  - member：有 + - × ÷ 按钮
  - admin/superadmin：同 member
- 所有计算结果需经后端验证（不能只在前端计算）
- 若用户尝试调用无权限的运算（如通过接口直接请求），返回明确的拒绝提示

### 3. 管理面板（admin/superadmin 可见）

管理员登录后，计算器界面下方出现管理面板：

- 列出所有用户及其当前角色
- 可将某用户**升级为 member** 或**降级为 guest**
- 可将某用户**封禁**（封禁后该用户无法登录，提示"账号已被封禁"）
- 解除封禁
- admin 不能修改其他 admin 或 superadmin 的角色

### 4. 超级管理员：篡改数学规则（核心特色功能）

superadmin 界面额外有一个「规则篡改」面板：

- 可以为特定计算设置「覆盖结果」，例如：
  - 设置 `1 + 1 = 3`
  - 设置 `9 / 3 = 0`
- 覆盖规则**仅在本次服务运行期间生效**（重启后恢复正常）
- 规则生效后，**所有用户**（包括 guest）计算该表达式时都会得到被篡改的结果
- 界面上展示当前生效的所有篡改规则列表，支持删除单条规则

**示例效果**：
> superadmin 设置 `2 + 2 = 5`
> alice（guest）计算 `2 + 2`，结果显示 `5`
> 计算界面不提示任何异常，就好像数学本来如此

### 5. 计算历史

- 每个用户登录后可查看自己的最近 10 条计算记录
- 记录包含：表达式、结果、时间
- 若结果被规则篡改，历史记录中存储的是**篡改后的结果**

## 技术要求

1. **代码必须分文件组织**，至少包含：
   - `server.py`：HTTP 服务器入口
   - `db.py`：数据库操作
   - `auth.py`：认证与权限逻辑
   - `calculator.py`：计算逻辑（含规则篡改引擎）
   - `static/index.html`：前端页面

2. 前端与后端通过 **JSON API** 通信（RESTful 风格）

3. 使用 **Cookie 或 token** 维持登录态（标准库实现即可）

4. 数据持久化到 `calc.db`（SQLite）

5. 错误处理：所有接口调用失败需返回清晰的错误信息，前端需展示，不能静默失败

---

## 验收场景

完成后，以下操作必须全部正确运行：

**场景 A：角色权限隔离**
1. 注册新用户 `alice`，登录后只看到 `+` 和 `-` 两个运算按钮
2. 用 `root` 登录，将 `alice` 升级为 member
3. `alice` 重新登录，现在可以看到 `×` 和 `÷` 按钮

**场景 B：权限封锁**
1. 封禁用户 `alice`
2. `alice` 尝试登录，提示"账号已被封禁"

**场景 C：数学规则篡改**
1. `root` 登录，设置规则 `2 + 2 = 5`
2. 注册新用户 `bob`（guest），登录后计算 `2 + 2`
3. 结果显示 `5`
4. `root` 删除该规则
5. `bob` 再次计算 `2 + 2`，结果恢复为 `4`

**场景 D：越权防御**
1. 通过浏览器开发者工具，直接向后端发送 `bob` 的乘法请求
2. 后端返回拒绝，前端显示错误提示
