# 多角色权限计算器 · 使用指南

一个运行在本地的 Web 计算器，不同角色看到不同按钮；超级管理员还能"篡改数学"。
纯 Python 标准库 + 原生 HTML/JS，零第三方依赖。

## 1. 环境要求

- **Python 3.8+**（推荐 3.10 及以上，只用标准库，不需 pip install）
- 任意现代浏览器（Chrome / Edge / Firefox / Safari）
- 端口 **8080** 未被占用

## 2. 快速开始

```bash
# 1. 进入项目目录
cd C:\Users\lyi\Desktop\claude-opus-4.7

# 2. 启动服务
python server.py

# 看到输出：
#   Calculator server running on http://localhost:8080
#   Default superadmin: root / root123
```

打开浏览器访问 <http://localhost:8080>，看到登录页即成功。

**默认超级管理员账号**：

| 用户名 | 密码 |
|--------|------|
| `root` | `root123` |

> 首次启动会自动创建 `calc.db`（SQLite）并写入 superadmin。后续启动会复用已有数据库。
> 想重置全部数据 → 停止服务、删除 `calc.db`、再启动即可。

**停止服务**：在运行 `python server.py` 的终端按 `Ctrl + C`。

## 3. 角色与权限

系统有 4 种固定角色，权限**严格分层**：

| 角色 | 加 `+` | 减 `−` | 乘 `×` | 除 `÷` | 管理用户 | 篡改数学 |
|------|:---:|:---:|:---:|:---:|:---:|:---:|
| **guest**（游客） | ✅ | ✅ | ❌ | ❌ | ❌ | ❌ |
| **member**（会员） | ✅ | ✅ | ✅ | ✅ | ❌ | ❌ |
| **admin**（管理员） | ✅ | ✅ | ✅ | ✅ | ✅（不能动 admin/superadmin）| ❌ |
| **superadmin**（超管） | ✅ | ✅ | ✅ | ✅ | ✅（除自己）| ✅ |

**注册 = guest**。升级为 member / admin 需要管理员操作。superadmin 账号无法通过界面创建。

## 4. 功能使用

### 4.1 注册与登录

**注册新账号**

1. 打开 <http://localhost:8080>
2. 点顶部「**注册**」标签页
3. 输入用户名 + 密码（均不能为空；用户名 ≤ 32 字符，密码 ≤ 128 字符）
4. 点「注册」→ 注册成功会自动登录，角色为 **guest**

**登录**

1. 顶部选「**登录**」标签
2. 输入用户名 + 密码 → 回车或点「登录」

**登出**

- 页面右上角 → 点「**登出**」按钮

**关于会话**
- 登录凭证存在 `HttpOnly` Cookie 中，关闭标签页不会立即失效，浏览器关闭也通常保留。
- 管理员修改你的角色或封禁你时，你的会话会被**立即作废**，下次操作时需要重新登录。

---

### 4.2 使用计算器

登录后看到「计算器」卡片：

```
┌─────────────────────────────┐
│            —                │   ← 显示区
├─────────────────────────────┤
│ [操作数 A]  ?  [操作数 B]   │
├─────────────────────────────┤
│  [+]  [−]  [×]  [÷]         │   ← 按钮随角色变化
└─────────────────────────────┘
```

**操作步骤**：

1. 左框输入数字 A（如 `2` 或 `3.14`）
2. 右框输入数字 B
3. 点击某个运算按钮 → 结果显示为 `A op B = 结果`

**支持的数字格式**：
- 整数：`1`、`42`、`-7`
- 小数：`3.14`、`0.5`
- 科学记数法：`1e3`（= 1000）

**按钮显示规则**：
- **guest** 只看到 `+` 和 `−`
- **member / admin / superadmin** 看到全部 4 个
- 看不到按钮 ≠ 没有权限检查：即使你绕过前端直接调 API，后端同样会按角色拦截（见 [场景 D](#场景-d越权防御)）

**常见错误提示**：
| 提示 | 原因 |
|------|------|
| 请填入两个操作数 | A 或 B 为空 |
| 无法解析数字: xxx | 输入不是合法数字 |
| 除数不能为零 | B = 0 且运算符为 `÷` |
| 当前角色（xxx）无权使用运算符 x | 越权（前端或直接调接口） |

---

### 4.3 查看计算历史

计算器下方的「**最近 10 条历史**」卡片显示：

| 列 | 含义 |
|---|---|
| 表达式 | 规范化后的表达式，如 `2 + 2` |
| 结果 | **最终呈现给用户的结果**（如果被篡改，这里就是篡改值） |
| 时间 | 本地时间 |

- 每次计算成功后自动刷新；也可点右上角「**刷新**」按钮手动刷新。
- 每个用户**只看到自己的历史**，管理员也看不到别人的。
- 历史保存在 SQLite 里，**服务重启后仍然存在**。

### 4.4 管理面板（admin / superadmin）

角色为 `admin` 或 `superadmin` 时，计算器下方出现「**管理面板**」：

```
管理面板
┌────┬─────────┬─────────┬──────┬─────────────────────────────┐
│ ID │ 用户名  │ 角色    │ 状态 │ 操作                        │
├────┼─────────┼─────────┼──────┼─────────────────────────────┤
│ 1  │ root    │superadm │ 正常 │ 无权操作（admin 视角）     │
│ 2  │ alice   │ guest   │ 正常 │ [guest▼] [保存角色] [封禁] │
│ 3  │ bob     │ member  │ 封禁 │ [guest▼] [保存角色] [解除封禁]│
└────┴─────────┴─────────┴──────┴─────────────────────────────┘
```

**可执行操作**：

1. **改角色**
   - 从下拉框选新角色 → 点「保存角色」
   - **admin 只能设置**：`guest` / `member`
   - **superadmin 可设置**：`guest` / `member` / `admin`
   - 界面不提供"设置为 superadmin"的选项；后端也会拒绝该请求
2. **封禁 / 解封**
   - 点「封禁」→ 目标用户立即被踢下线，之后登录会提示「账号已被封禁」
   - 点「解除封禁」→ 恢复登录能力
3. **无权操作**
   - 自己的那一行显示「当前账号」，不能改自己的角色也不能封自己
   - `admin` 看到其他 `admin` / `superadmin` 时显示「无权操作」

**权限改动即时生效**：被改角色或封禁的用户，其现有会话会被服务器废弃，必须重新登录。

### 4.5 规则篡改（superadmin）

`superadmin` 专属的「**规则篡改面板（仅本次服务生效）**」位于管理面板下方：

```
规则篡改面板（仅本次服务生效）
┌────┬────┬────┬───┬───────────┬──────────┐
│ A  │ op │ B  │ = │ 覆盖结果  │ [添加规则]│
└────┴────┴────┴───┴───────────┴──────────┘

当前生效规则：
┌───────────────┬──────────────┬─────────┬───────┐
│ 表达式        │ 真实结果应为 │ 被篡改为│ 操作  │
├───────────────┼──────────────┼─────────┼───────┤
│ 2 + 2         │ 4            │ 5       │ [删除]│
│ 9 ÷ 3         │ 3            │ 0       │ [删除]│
└───────────────┴──────────────┴─────────┴───────┘
```

**添加规则**：
1. 填入 A、运算符、B、以及你想要的"覆盖结果"
2. 点「添加规则」

**规则生效规则**：
- 对**所有用户**（包括 guest）生效
- 匹配时按**规范化后**的数值比较：`2 + 2` 和 `2.0 + 2.0` 视为同一条规则
- **仅在本次服务运行期间生效**：重启 `server.py` 后所有覆盖规则清空，数学恢复正常
- 覆盖规则**不影响按钮可见性**：guest 仍然只能做加减，但他们做的 `2+2` 会得到被篡改的 5

**删除规则**：点该行右侧「删除」按钮即可。

**不受影响的输入**：不在覆盖表里的表达式继续走真实数学。

## 5. 典型场景演练

### 场景 A：角色权限隔离

1. 注册 `alice` / `pw123` → 登录 → 计算器只显示 `+` `−`
2. 登出 → 用 `root` / `root123` 登录
3. 管理面板里找到 alice → 下拉选 `member` → 保存
4. 登出 → 用 alice 重新登录 → 计算器现在显示 `+ − × ÷` 四个按钮 ✅

### 场景 B：权限封锁

1. root 登录 → 管理面板中封禁 alice
2. 用 alice 登录 → 提示「账号已被封禁」 ✅

### 场景 C：数学规则篡改

1. root 登录 → 规则篡改面板 → 添加 `2 + 2 = 5`
2. 注册 / 登录 `bob`（guest）
3. 输入 A=`2`、B=`2`，点 `+` → 显示 `2 + 2 = 5` ✅
4. 登出 → root 登录 → 删除该规则
5. bob 再次计算 → 恢复为 `4` ✅

### 场景 D：越权防御

即使通过浏览器开发者工具直接向后端发乘法请求：

```js
fetch('/api/calc', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({ a: '3', op: '*', b: '4' })
}).then(r => r.json()).then(console.log)
```

作为 guest 的 bob 会收到：

```json
{ "ok": false, "error": "当前角色（guest）无权使用运算符 *" }
```

HTTP 状态 **403 Forbidden** ✅


## 6. 接口一览（高级用户 / 调试）

所有接口均返回 JSON，字段约定：
- 成功：`{"ok": true, ...}`
- 失败：`{"ok": false, "error": "错误描述"}`（配合非 2xx 状态码）

会话通过 Cookie `session` 传递，`HttpOnly + SameSite=Lax`。

### 认证

| 方法 | 路径 | 权限 | Body | 说明 |
|---|---|---|---|---|
| POST | `/api/register` | 无 | `{username, password}` | 注册（默认 guest） |
| POST | `/api/login`    | 无 | `{username, password}` | 登录，Set-Cookie |
| POST | `/api/logout`   | 已登录 | `{}` | 清会话 |
| GET  | `/api/me`       | 任意 | — | 返回 `{user: ...}` 或 `{user: null}` |

### 计算与历史

| 方法 | 路径 | 权限 | Body | 说明 |
|---|---|---|---|---|
| POST | `/api/calc` | 已登录 + 角色允许该运算符 | `{a, op, b}` | 返回 `{expression, result}` |
| GET  | `/api/history` | 已登录 | — | 最近 10 条 |

### 用户管理

| 方法 | 路径 | 权限 | Body |
|---|---|---|---|
| GET  | `/api/users`        | admin / superadmin | — |
| POST | `/api/users/role`   | admin / superadmin | `{user_id, role}` |
| POST | `/api/users/ban`    | admin / superadmin | `{user_id, banned: true\|false}` |

约束：
- 不能修改自己
- `admin` 不能动 `admin` / `superadmin`
- `role` 只能取 `guest` / `member` / `admin`（设成 `superadmin` 会被拒）

### 规则篡改

| 方法 | 路径 | 权限 | Body |
|---|---|---|---|
| GET  | `/api/overrides`           | superadmin | — |
| POST | `/api/overrides`           | superadmin | `{a, op, b, result}` |
| POST | `/api/overrides/delete`    | superadmin | `{key}` |

`key` 从 `GET /api/overrides` 返回的每条规则中获取。

### 用户对象结构

```json
{
  "id": 2,
  "username": "alice",
  "role": "member",
  "banned": false,
  "allowed_ops": ["+", "-", "*", "/"],
  "can_manage": false,
  "can_tamper": false
}
```

## 7. 常见问题 FAQ

**Q1：端口 8080 被占用怎么办？**
编辑 `server.py` 顶部的 `PORT = 8080`，改成空闲端口（如 `8090`），重启即可。

**Q2：忘了 root 密码？**
停止服务 → 删除 `calc.db` → 重启服务。**注意这会清掉所有用户、历史**。
若只想重置 root：可以用任意 SQLite 工具更新 `users` 表中 `root` 的 `pw_hash` / `pw_salt`（参考 `auth.hash_password("新密码")`）。

**Q3：为什么我改了 alice 的角色，她还是看不到新按钮？**
角色变更会**作废她的旧会话**。她需要**重新登录**，前端再次拉取 `/api/me` 才能看到新按钮。

**Q4：规则篡改重启后丢了？**
这是设计要求。规则只存内存，重启即清空。如需持久化，可以自行改 `calculator.py` 把 `_overrides` 写进数据库（不推荐，破坏"本次服务运行期间生效"的语义）。

**Q5：可以同时打开多个浏览器 / 多个账号吗？**
可以。不同浏览器 / 隐身窗口有各自的 Cookie，互不干扰。

**Q6：为什么 `1 + 1` 的结果显示 `2` 而不是 `2.0`？**
`calculator.format_result()` 会把整数值的浮点数格式化成整数显示，提升可读性。

**Q7：我想要 100 条历史而不是 10 条？**
编辑 `db.recent_history(user_id, limit=10)` 的默认值，或在 `server.py._api_history` 中调用时传更大值。

**Q8：数据库文件在哪？**
与 `server.py` 同目录的 `calc.db`（SQLite 单文件）。可以用 DB Browser for SQLite、DBeaver 等工具直接打开。

**Q9：服务会自动过期会话吗？**
当前实现**不主动过期**（除非用户被封禁 / 改角色 / 主动登出）。如需加 TTL，可在 `auth.current_user()` 内用 `sessions.created_at` 判断时长并清理。

**Q10：可以在局域网里让别人访问吗？**
默认已监听 `0.0.0.0:8080`，局域网内其他机器访问 `http://<你的IP>:8080` 即可。注意：**本项目用明文 HTTP，仅适用于本地 / 受控环境，不要暴露到公网**。

## 8. 项目结构

```
C:\Users\lyi\Desktop\claude-opus-4.7\
├── server.py          # HTTP 入口 + 路由 + JSON API
├── db.py              # SQLite 表结构与 DAO
├── auth.py            # 密码哈希、会话、权限矩阵
├── calculator.py      # 二元运算 + 内存态规则覆盖
├── smoke_test.py      # 端到端自动化验收脚本
├── static/
│   └── index.html     # 单页前端（纯 HTML/CSS/JS）
└── calc.db            # SQLite 数据（首次启动自动生成）
```

各模块职责清晰，互不越界：
- **`db.py`** 只管表与 CRUD，不涉权限
- **`auth.py`** 不碰 HTTP，纯函数式权限判断
- **`calculator.py`** 不碰数据库，把历史落库交给上层
- **`server.py`** 做路由 + 权限门控 + 参数校验，调下层模块
- **`index.html`** 把服务器返回的 `allowed_ops` / `can_manage` / `can_tamper` 映射到 UI
